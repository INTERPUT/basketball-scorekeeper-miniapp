## 一、3 分钟 Demo 视频方案

### 视频目标

用一个完整业务故事证明项目能解决真实技术台痛点：专业设施成本高、基层翻分牌/记录表难用、按钮式计分仍有操作门槛；本项目用自然语言录入、规则提醒、观众实时查看、赛后签字和中文报告归档完成低门槛闭环。

### 推荐录制环境

- 录屏工具：Windows 自带截图工具 / Xbox Game Bar / OBS 均可。
- 录制对象：微信开发者工具模拟器 + 必要时切换到项目 README / GitHub 页面。
- 分辨率：建议 1080p。
- 时长：控制在 2 分 40 秒到 2 分 55 秒，留 5-10 秒余量。
- 输入方式：只演示自然语言文本输入，不演示语音，避免不稳定因素。

### 建议演示数据

- 房间码：录制前临时生成一个未用四位码。
- 主队：白队，球员 `7 张三`、`11 赵六`。
- 客队：蓝队，球员 `8 李四`、`12 王五`。
- 比赛名称：黑客松 Demo 赛。
- 场地：微信开发者工具模拟器。
- 裁判：i坤队。

### 视频时间轴

| 时间 | 画面 | 旁白重点 |
|---|---|---|
| 0:00-0:35 | HTML-PPT 痛点页 | 专业计分设备贵且复杂；基层赛事依赖翻分牌和记录表，观众看分难、技术台容易漏记 |
| 0:35-0:55 | 现有产品不足页 | 按钮式小程序仍要找按钮、切模块，暂停、时间和犯规临界提醒不足 |
| 0:55-1:15 | 建赛页 | 创建比赛，添加主客队球员，展示“添加球员”弹窗 |
| 1:15-1:30 | 加入页 + 观众页 | 观众输入四位房间码，进入只读比分页 |
| 1:30-2:05 | 技术员页自然语言录入 | 输入并确认事件，展示比分、犯规、暂停和提醒变化 |
| 2:05-2:25 | 跨节和比赛结束 | 输入“结束本节比赛”“开始计时”“比赛结束”，展示 finished 状态 |
| 2:25-2:45 | 签字归档页 | 双方签字，生成归档，展示 HTML / JSON / PDF 文件 |
| 2:45-2:55 | 打开 PDF 报告 + GitHub | 展示中文报告、球队得分、球员个人得分；结尾给出仓库和队伍名 |

### 推荐录入事件

为了 3 分钟内完成，视频只需演示缩短版：

```text
开始计时
白队 7 号两分命中
蓝队 8 号三分命中
白队暂停
蓝队 8 号犯规一次
结束本节比赛
开始计时
白队 11 号三分命中
比赛结束
```

口播补充：完整 4 节、19 条事件的自动化场景已通过，最终比分白队 8:6 蓝队，并已生成包含球队得分和球员个人得分的中文 PDF 报告。
