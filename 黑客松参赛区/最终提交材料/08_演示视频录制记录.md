## 覆盖功能

- 痛点说明：专业计分设施贵且复杂，基层篮球赛依赖翻分牌和记录表，观众看分难、技术台容易漏记。
- 现有产品不足：按钮式小程序仍需找按钮、切模块，暂停、时间和犯规临界提醒不足。
- 产品定位：面向校园篮球爱好者比赛，技术员只需清楚描述场上事件，系统转为得分、犯规、暂停和提醒。
- 创建比赛：四位房间码、队伍信息、添加球员弹窗。
- 观众加入：通过房间码进入只读比分页。
- 自然语言录入：解析待确认事件，确认后入账。
- 比赛状态更新：比分、逐节比分、犯规、暂停、事件流水。
- 比赛结束：展示 finished 状态与最终比分。
- 双方签字：白队、蓝队队长签字确认。
- 归档导出：HTML 计分表、JSON 数据、中文 PDF 报告。
- 中文报告：包含球队得分、球员个人得分和犯规统计。
- Vibe Coding：展示测试通过、Debug 修复和首版完成度。

## 录制方式

本轮根据修改要求，先使用 `archlizheng/frontend-slides-editable` 的 HTML-PPT 技能方案生成可编辑演示页，再用 PowerPoint 导出 MP4：

1. 克隆并参考 `frontend-slides-editable` 的 `SKILL.md` 和 `viewport-base.css`。
2. 生成 11 页单文件 HTML-PPT，包含真实场景素材、基层技术台示意图、产品截图和测试截图。
3. 使用 Chrome headless 按页导出 1920x1080 PNG 帧，并生成 `html-ppt_contact_sheet.png` 做视觉检查。
4. 通过 PowerPoint COM 生成视频源 PPTX。
5. 使用每页自动播放时长导出 1080p MP4。

## Bug 与 Debug 记录

- 录制前功能回归：`npm test` 通过，6 个测试文件、21 个测试用例全部通过。
- 录制前类型检查：`npm run typecheck` 通过。
- HTML-PPT 截图问题：首次使用滚动定位时，Chrome headless 截图会串页。已改为 `?shot=N` 模式下只渲染目标 slide，再重新导出 11 张帧。
- 中文标题排版问题：初版字号过大，长中文标题换行挤压。已降低标题最大字号，并对第 8、9 页手动断行。

- 小程序功能：本次录制过程中未发现新的功能回归；视频依据已通过的完整自然语言比赛测试与最终归档结果制作。
