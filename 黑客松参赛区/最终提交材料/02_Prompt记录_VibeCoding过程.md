# Prompt 记录与 Vibe Coding 过程

## 1. Vibe Coding 总体方式

本项目采用“需求文档先行 + AI 连续实现 + 小步验证 + 状态文件接力”的方式推进。AI 不是只生成静态页面，而是参与完整工程闭环：读取需求、拆阶段、写代码、接云开发、部署云函数、执行自动化测试、根据模拟器反馈 debug、整理最终提交材料。

## 2. 核心 Prompt 记录

### Prompt 1：建立唯一需求基线并进入 Goal 模式

```text
请先完整阅读以下四份文档，并把它们视为当前项目的唯一需求基线。
你的任务不是只给建议，而是直接在当前工作区持续完成这个微信小程序，直到达到文档定义的首版目标。
以《技术实现方案与开发任务拆解》为开发基线，按其中阶段顺序推进。
每完成一个阶段，都要运行可运行的验证，并继续推进下一阶段。
```

作用：让 AI 从“建议者”转成“持续执行的工程代理”，并建立需求、实现、测试和状态记录之间的闭环。

### Prompt 2：真实云开发接线

```text
下一步请直接进入真实接线：
1. 在小程序侧初始化云开发环境
2. 把本地仓储替换为真实云数据库 + 实时监听
3. 创建 parseEvent / confirmEvent / transcribeAudio 等云函数
4. 把火山 ASR 凭据放到云函数私有配置或环境变量，不要写进前端
5. 做一次云数据库、云存储、ASR 的真实端到端联调
```

作用：推动原型从本地假数据变成真实微信云开发链路。

### Prompt 3：根据模拟器反馈做产品修正

```text
1. 直接进入一个不存在比赛，报错不应该是 document get fail，改成未找到比赛房间。
2. 球员名单的输入，不符合规范就报错，我想把球员名单改成添加球员的形式。
3. 你自行测试并继续改进，直到完成目标。
```

作用：让 AI 在真实使用反馈下修正体验，不停留在技术正确。

### Prompt 4：自然语言完整比赛测试

```text
先不用语音功能，就自然语言输入开始，你模拟一个完整比赛测试这个小程序，然后 debug，直到小程序能通过完整的使用场景测试。
```

作用：把验收标准从单点功能提升到完整业务场景，最终发现并修复了跨节解析和观众页初始快照问题。

### Prompt 5：最终报告中文化

```text
最后的报告，需要输出中文，然后除了球队得分，加入球员个人得分内容。
```

作用：让赛后导出从技术证明变成可提交、可读、可交付的中文报告。

## 3. AI 直接生成或主导的内容

- 原生微信小程序工程骨架。
- TypeScript 领域模型：`Match`、`Room`、`Team`、`Player`、事件类型、归档类型。
- 建赛、加入、技术员、观众、签字归档页面。
- 自然语言解析规则和事件草稿契约。
- 事件确认、事件回放、比分和犯规统计规则。
- 云函数：建赛、解析、确认、签字、归档、状态恢复、ASR。
- 自动化测试：规则引擎、事件解析、房间服务、云错误归一。
- 微信开发者工具自动化场景脚本。
- 最终提交文档、README、演示脚本。

## 4. 人工决策和手动优化

- 决定以自然语言文本输入作为首版验收主线，语音先作为增强项。
- 决定用“添加球员”弹窗替代多行文本名单，降低非技术用户输入成本。
- 决定导出报告必须中文化，并加入球员个人得分，便于比赛后交付和评审展示。
- 提供真实小程序 AppID、云开发 envId、ASR 凭据，并完成微信开发者工具登录。
- 在模拟器中反馈真实使用问题，推动 AI debug。

## 5. 关键 Debug 记录

| 问题 | 现象 | 解决 |
|---|---|---|
| 不存在房间暴露底层错误 | 显示 `document.get:fail cannot find document...` | 在云服务层归一为“未找到比赛房间。” |
| 球员名单格式不友好 | 用户需要按“号码 姓名”多行输入 | 改为“添加球员”弹窗，分别输入号码和姓名 |
| 观众页加入后空态 | 只开监听，没有先读当前快照 | `watchRoomSnapshot` 先拉当前快照，再开启监听 |
| 自然语言无法进入下一节 | 上一节结束后再“开始计时”仍是当前节 | 解析器根据 `period_ended` 自动把 period + 1 |
| PDF 报告英文且信息不足 | PDF 只显示英文摘要和队伍总分 | 嵌入中文字体，输出中文报告，加入个人得分/犯规表 |

## 6. Vibe Coding 亮点

- AI 按阶段持续推进，而不是只生成片段。
- 状态文件 `PROJECT_STATUS.md` 保证中途接力可继续。
- 每个阶段都有可运行验证，避免“看起来完成但不能跑”。
- 真实云函数部署和真实云端数据验证纳入开发过程。
- 最终用完整比赛场景自动化来定义“首版完成”。

## 7. 可用于提交的过程总结

本项目的 Vibe Coding 不是单次生成代码，而是一次多轮协作式工程实践：先将产品、事件字典、页面流程和技术方案作为唯一需求基线，再让 AI 按阶段实现，过程中持续运行类型检查、单元测试、云端联调和微信开发者工具自动化测试。用户通过模拟器反馈真实体验问题，AI 继续修正并重跑完整场景，最终完成一个可运行、可演示、可归档的微信小程序首版。
